Authentication: All live endpoints require a Bearer token. You must obtain your JWT access token from the login endpoint and include it as a header in every Power Query request.

Step-by-Step: Connect Excel via Power Query

1

Open Excel → Click the Data tab on the ribbon.

2

Click Get Data → From Other Sources → From Web.

3

Select Advanced and enter the URL:

http://[your-server]:4000/api/v1/live/ledger

Replace [your-server] with the server's IP or hostname.

4

In the HTTP request header parameters section, add:

Authorization: Bearer <your-jwt-token>

Obtain your token by calling POST /api/v1/auth/login with your emp_no and password. The access_token field in the response is your Bearer token.

5

Click OK. Power Query will fetch the data. In the Power Query Editor that opens, the JSON array is automatically converted to a table. Click Close & Load to load it into a sheet.

6

Set auto-refresh: Go to Data → Queries & Connections. Right-click your query → Properties. Under Usage, check Refresh every X minutes and enter your preferred interval (e.g., 15). Also check Refresh data when opening the file.

OneDrive Auto-Export

When a declaration is approved through the full chain and reaches COMPLETED status, the system automatically generates an Excel report and uploads it to the configured SharePoint/OneDrive folder. Facility managers can also trigger a manual upload from the Live Excel page in the web app.

The shared OneDrive file is always current — no need to download manually. Open the shared link in your browser and click Edit in Browser or Open in Desktop App to access the latest data.
